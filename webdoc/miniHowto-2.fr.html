2. Installation de GNU/ldapcppei :

2.1 Où récupérer les sources de GNU/ldapcppei ?

L'interface objet se présente sous forme de sources compilables contenues dans un paquet logiciel qui se trouve ici et se présente sous la forme d'un fichier d'archivage au format "tar" compressé "gzip" (".tgz").
Il a pour nom : "libldapcppei-$(numéro_de_version).tgz".
(Le paquet logiciel "libldap2-dev" est également nécessaire).

2.2 Comment préparer la compilation de GNU/ldapcppei ?

Après téléchargement du paquet logiciel (par exemple: dans le répertoire "/tmp"), taper :

...:~$ cd /tmp
...:/tmp/$ tar xzf libldapcppei-$(version).tgz
...:/tmp/$ cd libldapcppei/

(Nota: Un fois les opérations de compilation et d'installation - détaillées ci-après - effectuées, ce répertoire "libldapcppei/" et tout son contenu pourra être supprimé par la commande "rm -r ...").

2.3 Comment compiler GNU/ldapcppei ?

Dans ce même répertoire (ici : "/tmp/libldapcppei/"), taper la commande "make". Elle va construire, dans le sous répertoire "./src/", les trois bibliothèques suivantes :

  • libldapcppei.so : bibliothèque pour une édition de lien dynamique,
  • libldapcppei-dbg.so : contient les informations de déboguage (parfois utile aux développeurs ;-),
  • libldapcppei.a : bibliothèque pour une édition de lien en statique (code intégré à l'exécutable du programme qui l'utilisera ...).

2.4 Comment installer GNU/ldapcppei ?

Exécuter la commande "sudo make install" ; ainsi, les 3 bibliothèques précédemment générées seront déployées dans leur arborescence d'accueil ("/usr/local/lib") et leur emplacement sera déclaré aux applications utilisatrices via le sous-système "ldconfig".

Les fichiers "include.h" seront copiés dans l'arborescence "/usr/local/include/" (par défaut).

2.5 Comment déployer les sources de GNU/ldapcppei ?

Taper la commande "sudo make installsrc" ; celle-ci aura pour effet de mettre à disposition les sources ("*.cpp") de la bibliothèque dans le répertoire d'accueil "/usr/local/src/ldapcppei/" (par défaut).

2.6 Comment désinstaller GNU/ldapcppei ?

La commande "sudo make uninstall" permet de désinstaller les bibliothèques précédemment installées, efface les fichiers "include.h" propres à l'interface ainsi que les fichiers sources éventuellement installés et leur répertoire de localisation ("/usr/local/src/ldapcppei/").

2.7 Comment obtenir le paquet DEBIAN ?

La commande "make debian" (nécessite le paquet logiciel "fakeroot") permet d'obtenir le paquet "DEBIAN.deb" permettant de distribuer la bibliothèque (avec ses sources et gestion des dépendances) ...

NOTA: L'installation précédemment décrite (2.3 - 2.6) ne s'applique plus dans ce cas précis puisqu'il suffit en effet d'installer tout simplement le paquet ("*.deb") ainsi obtenu ; l'installation peut alors être réalisée avec l'un des utilitaires (en mode commande ou mode graphique) gérant ce type de distribution (ex.: "sudo apt-get install *deb").
