2. Installing the GNU/ldapcppei API:

2.1 Where to get the sources of the GNU/ldapcppei API?

The API is in the form of compilable sources contained in a software package that is here and comes in the form of an archive file format "tar" compressed "gzip" (".tgz").
Its name is: "libldapcppei-$(version_number).tgz".
(The "libldap2-dev" package is also needed).

2.2 How to prepare the compile of the GNU/ldapcppei API?

After downloading the package (for example: to the directory "/tmp"), type:

...:~$ cd /tmp
...:/tmp/$ tar xzf libldapcppei-$(version).tgz
...:/tmp/$ cd libldapcppei/

(Note: Once operations build and installation finished (- detailed below -), this directory "libldapcppei/" and all its contents could be deleted by the command: "rm -r ...").

2.3 How to compile the GNU/ldapcppei API?

In this same directory (here: "/tmp/libldapcppei/"), type the commande "make". It will build, in the subdirectory "./src/", the three following libraries:

  • libldapcppei.so : library for dynamic linking,
  • libldapcppei-dbg.so : contains the debugging information (sometimes useful to developers ;-),
  • libldapcppei.a : library for linking in static mode (embedded code to the executable program to use ...).

2.4 How to install the GNU/ldapcppei API?

Run the command "sudo make install"; so the previously generated three libraries will be deployed in their host tree ("/usr/local/lib") and their location will be declared to user applications via the subsystem "ldconfig".

Files "include.h" will be copied in the tree "/usr/local/include/" (default).

2.5 How to deploy the sources of the GNU/ldapcppei API?

If needed, run the command "sudo make installsrc"; it will effectively make available the sources ("*.cpp") of the library in the home directory "/usr/local/src/ldapcppei/" (default).

2.6 How to uninstall GNU/ldapcppei ?

The command "sudo make uninstall" allows to uninstall the librairies previously installed, to delete the "include.h" specific files, source files eventually installed and the default directory location ("/usr/local/src/ldapcppei/") of this package.

2.7 How to get the DEBIAN package?

The command "make debian" (needs the "fakeroot" package installed) allows to get the "DEBIAN.deb" (containing sources and dependencies management)...

NOTE: The installation described above (2.3 - 2.6) no longer applies in this case since it suffices to simply install the package ("*.deb") thus obtained; the API can then be performed with one of utilities (command mode or graphics mode) managing this type of distribution (eg "sudo apt-get install *.deb").
