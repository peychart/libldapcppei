15.2  The systems setup interface "doCronConf"

Here is the "crond" setup interface: doCronConf. Code is shared with getHostName. It can be used as follow:
                getHostName  --cronConf
or:          ln  -s  /usr/local/bin/getHostName  /usr/local/bin/doCronConf  ;  doCronConf

This program allows the automatic generation (either triggered by "cron", either at start-up procedures of the destination system) of the "crontabs" definitions for the "crond" daemonfrom the information (if any) contained in the LDAP database according to the MAC address of the current server (where the command is issued):
doCronConf -v -U ldap://ldap...

Example for shell start (/etc/init.d/) for each of the intranet servers to administrate:
       ...
       # previous setting...
       ...
       TMP=/tmp/.doCronConf.$$
       2>/dev/null ${BIN}/getHostName --cronConf  >$TMP
       [ $? -eq 0 ] && /bin/sh $TMP
       rm -f  $TMP
       ...
       # next setting...
       ...

Example of using this interface on the "test database":
root@me:~/Cdev#  doCronConf  -v --forceMacAddress  11:22:33:44:55:66
[ ! -d "/var/spool/cron/scripts/openldap" ] && mkdir -p "/var/spool/cron/scripts/openldap"
